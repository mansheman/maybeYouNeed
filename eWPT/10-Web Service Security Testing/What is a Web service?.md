# Web Services

## What Are Web Services?

- Software components that enable communication and data exchange between different applications/systems over the internet.
    
- Allow interoperability across platforms, programming languages, and servers.
    
- Main purposes:
    
    - **Integration between applications** (e.g., App A uses features of App B).
        
    - **Separation within an application** (e.g., front-end calling a backend service to update content).
        

---

## Web Services vs Web Applications

|Aspect|Web Services|Web Applications|
|---|---|---|
|**Purpose**|Facilitate data exchange between applications.|Provide services/tasks directly to end-users.|
|**User Interaction**|No UI; machine-to-machine.|User-friendly interface for humans.|
|**Data Exchange**|Structured data (SOAP, REST, XML, JSON).|User input, processing, and presentation.|
|**Protocols**|SOAP, REST, XML-RPC, JSON-RPC.|Primarily HTTP/HTTPS.|
|**Security Focus**|Data transmission security, access control.|Authentication, authorization, data validation, web security.|
|**Examples**|PayPal API, weather data services.|Gmail, Amazon, Facebook.|

---

## Key Characteristics of Web Services

- **Interoperability:** Standardized communication using open standards (HTTP, SOAP, REST, JSON, XML).
    
- **Platform-agnostic:** Independent of OS or programming language.
    
- **Loose Coupling:** Changes in one system don’t break others.
    
- **Location Independence:** Can be hosted anywhere and accessed via the internet.
