# Session IDs & Cookies

## Overview
- Both are critical for **session management** and **user authentication**.  
- Common focus areas in web application penetration testing.  

## Session IDs
- Unique tokens generated by the server to identify and track user sessions.  
- Maintain **stateful communication** between client and server.  
- Associate user requests with their session data stored on the server.  

**Example**  
- After login, server generates a session ID (e.g., `Session12345`).  
- The ID is tied to the user’s session and sent to the browser as a cookie.  

## Cookies
- Small pieces of data (text) stored in the user’s browser.  
- Purposes:  
  - Session management  
  - User tracking  
  - Personalization  
- **Session cookies** usually store the session ID.  

**Example**  
- On login, server sends cookie:  
	sessionID = Session12345

- Browser includes this cookie in subsequent requests.   
- Server uses it to identify and maintain the session.`
